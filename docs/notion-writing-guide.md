# Notion 작성 가이드 (팀원용)

이 사이트는 Notion Posts DB에서 **Status가 Published인 글만** 수집해 배포합니다.
글 작성 시 아래 규칙을 지켜주세요.

## 필수 속성

| 속성 | 규칙 |
|---|---|
| Title | 게시글 제목. 비어 있으면 빌드 실패 |
| PostId | 영문 소문자와 하이픈만. 예: `pr-management`. URL과 파일명이 됨 |
| Module | select에서 선택: OV / FI / CO / MM / SD / PP / RP |
| Status | `Draft`(작성 중) → `Published`(공개). **Published로 바꾸는 순간 배포 대상** |
| Order | 모듈 안 정렬 순서. 10, 20, 30처럼 10 단위 권장 |
| Summary | 목록 카드에 보이는 한 줄 요약. 비어 있으면 빌드 실패 |

## 이미지 사용 방법

**빌드할 때 Notion 이미지는 자동으로 다운로드되어 사이트에 저장됩니다.**
Notion 임시 URL이 만료되어도 사이트 이미지는 깨지지 않습니다.

- **ScreenUrls 속성**: 스크린샷 URL을 줄바꿈 또는 쉼표로 구분해 입력.
  - Notion에 업로드한 이미지 URL도 됩니다(자동 로컬화됨).
  - 외부 이미지 URL(예: GitHub raw)은 그대로 사용됩니다.
- **ThumbnailUrl 속성**: 대표 이미지 1장. 같은 방식으로 처리됩니다.
- **본문에 붙여넣은 이미지**: 자동으로 다운로드는 되지만 아직 화면에는 표시되지 않습니다(추후 지원).

### 이미지 주의사항

- **SVG는 지원하지 않습니다** (보안상 제외되고 경고만 남습니다).
- 이미지 1장은 **10MB 이하**, 게시글 전체는 **50MB 이하** 권장. 초과하면 빌드 로그에 경고가 남습니다.
- 권장 크기: 가로 1200~1600px, JPG/PNG/WebP.
- YouTube/Vimeo/Drive 링크는 ScreenUrls가 아니라 **VideoUrl** 속성에 넣으세요.

## Published로 바꾸기 전 체크리스트

1. Title / PostId / Module / Order / Summary가 채워져 있는가
2. 본문에 기능 개요·업무 흐름·구현 포인트·트러블슈팅 섹션(제목 블록)이 있는가
3. 스크린샷에 **회사 내부 정보가 없는가** (아래 보안 규칙)
4. 이미지가 잘 열리는가 (깨진 이미지는 빌드 실패 원인)

## 보안 규칙 — 스크린샷/본문 공통

다음이 보이는 화면은 올리지 마세요. 필요하면 마스킹 후 업로드하세요.

- 교육기관/회사의 서버 주소, IP, 시스템 ID
- SAP 계정, 사번, 실명(본인 동의 없는 타인 실명 포함)
- 실제 고객사명, 실거래 데이터, 전표번호
- Notion API Key, Netlify URL 등 운영 정보

## 반영 시점

Published로 바꿔도 **사이트에 바로 반영되지 않습니다.**
재배포(Build Hook 실행 또는 git push)가 필요합니다. 담당자에게 요청하거나 `02_deploy_portfolio.bat`을 실행하세요.
